修改 main.py 的最后部分（第 75-95 行），添加 session 持久化支持：

1. 在 deferred imports 区块增加导入：
   ```python
   from azure.ai.agentserver.agentframework.persistence import JsonLocalFileAgentSessionRepository
   import asyncio
   ```

2. 在 `agent = chat_client.as_agent(...)` 之后、`from_agent_framework(agent).run()` 之前，添加 session_repository 配置逻辑：
   ```python
   # 8. Configure session persistence
   session_repo = None
   if os.environ.get("ENABLE_SESSION_PERSISTENCE", "").lower() == "true":
       storage_path = os.environ.get("SESSION_STORAGE_PATH", "./thread_storage")
       session_repo = JsonLocalFileAgentSessionRepository(storage_path=storage_path)
       logger.info("Session persistence enabled: %s", storage_path)
   ```

3. 修改启动方式，把同步 `run()` 改为异步 `run_async()`，并传入 session_repository：
   ```python
   # 9. Start HTTP service
   logger.info("Starting SkillableAgent on http://localhost:8088")
   app = from_agent_framework(agent, session_repository=session_repo)
   asyncio.run(app.run_async())
   ```

注意：当 session_repo=None 时，SDK 的行为和改动前一致 —— 
在云端部署时 SDK 自动检测 AZURE_AI_PROJECT_ENDPOINT 并使用 FoundryConversationSessionRepository（per CONV-02）。
